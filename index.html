将实验报告呈现在网站上

编辑本地 Hexo 项目文件

在本地计算机上，打开 Hexo 项目文件夹。这里通常包含了所有的博客文章、配置文件和主题文件。
编辑文章：想要修改现有的文章，可以直接编辑 source/_posts 目录下对应的 Markdown 文件。使用文本编辑器打开文件，进行所需的修改，然后保存。
添加新文章：如果您想要添加新的文章，可以在 source/_posts 目录下创建一个新的 Markdown 文件。使用 hexo new “文章标题” 命令可以快速创建一个新的文章草稿。
修改主题和布局：如果想要更改网站的外观或布局，需要编辑 themes/使用的主题名 目录下的文件。这可能包括修改 HTML 模板、CSS 样式表或 JavaScript 文件。

生成静态文件

在本地对项目文件进行修改后，需要重新生成静态文件。在项目的根目录下打开命令行或终端，运行以下命令：

hexo clean && hexo generate
这将清除之前的生成文件，并创建新的静态文件到 public 目录。

部署到GitHub Pages

生成静态文件后，需要将这些更改推送到 GitHub 以更新网站。在项目的根目录下运行以下命令：

hexo deploy

之后就可以直接访问https://shixuanw.github.io/查看实验报告。

结论分析与体会

通过本次实验，我成功地熟悉了主流云平台的使用，并在 GitHub Pages 上搭建了个人博客网站。整个过程不仅加深了我对云计算技术的理解，还提升了我的实际操作能力。

我成功安装了 Node.js 环境，并通过 npm 全局安装了 Hexo，这是构建静态网站的基础。我利用 Hexo 快速搭建了博客的基本框架，在本地成功生成和预览了静态网页。并且还掌握了 Git 的基本操作，包括生成 SSH 密钥、添加远程仓库、提交更改和推送到 GitHub。学习了如何配置 Hexo 的部署工具，将本地的博客网站部署到了 GitHub Pages，实现了在线访问。

在实验过程中，我遇到了分支同步的问题，导致推送到 GitHub 时出现冲突。通过学习 Git 的合并和拉取操作，我成功解决了这一问题。对于 Hexo 主题和布局的修改，最初感到有些困难。但在查阅文档和参考社区的解决方案后，我逐渐理解了如何自定义网站的外观。

这次实验让我认识到了云平台在现代 web 开发中的重要性。未来，我计划继续深入学习云计算和 web 开发相关的技术，探索更多的云服务功能，提升我的开发能力。同时，我也希望能够通过不断实践，将我的博客网站打造得更加专业和个性化，成为我技术分享和个人品牌建设的平台。
